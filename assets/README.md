# Assets Folder

This folder is for storing project assets like 3D models, textures, and images.

## Current Setup

The portfolio currently uses **procedural geometry** (shapes generated by Three.js code), so no model files are needed. This is intentional for:

- Fast loading times
- No file dependencies
- Easy customization
- Minimal bundle size

## Adding Custom 3D Models (Optional)

If you want to use custom 3D models instead:

### 1. Supported Formats

- **GLTF/GLB** (recommended) - Best for web, compact
- **FBX** - Requires FBXLoader
- **OBJ** - Requires OBJLoader

### 2. Install Loaders

```bash
# GLTFLoader is included in Three.js examples
# No extra installation needed
```

### 3. Load Models

Example in `js/three-scene.js`:

```javascript
import { GLTFLoader } from 'three/examples/jsm/loaders/GLTFLoader.js';

// In createGeometry method:
const loader = new GLTFLoader();
loader.load(
  '/assets/models/your-model.glb',
  (gltf) => {
    this.mesh = gltf.scene;
    this.scene.add(this.mesh);
  },
  (progress) => {
    console.log('Loading:', (progress.loaded / progress.total * 100) + '%');
  },
  (error) => {
    console.error('Error loading model:', error);
  }
);
```

## Optimization Tips

1. **Keep models under 5MB** for fast loading
2. **Use compressed textures** (JPG over PNG for photos)
3. **Reduce polygon count** for web use (aim for <50k triangles)
4. **Use Draco compression** for GLTF files
5. **Test on mobile** to ensure performance
